## 3. Архитектурные правила

1. **Cargo workspace** — `Cargo.toml` в корне (virtual manifest). Каждая работа = отдельный crate в `crates/`.
2. **Clean Architecture** в каждом крейте:
   - `domain/` — чистые типы и алгоритмы, без `std::fs`, `std::io`, `tracing`.
   - `application/` — usecases, оркестрирует доменом через trait-абстракции.
   - `infrastructure/` — реализации портов (файлы, IO).
   - `presentation/` — CLI (`clap`).
3. **Зависимости**: разрешены `num-bigint`/`num-integer`/`num-traits` для длинной арифметики
   (условия данного курса не запрещают, в отличие от PSIA лаб 1).
   Алгоритмы (Рабин-Миллер, обмен ключами, СОК) пишем сами поверх готовой длинной арифметики.

## 4. Стек

- Rust **edition 2024** (rust-version = 1.85+).
- Логи: `tracing` + `tracing-subscriber` (pretty), инициализация через `shared::logging::init()`.
  Уровень — переменная `RUST_LOG`, по умолчанию `INFO`.
- Ошибки: `color-eyre::Result` в CLI; `thiserror` в доменных модулях.
- Тесты: `cargo test`, для криптографии — `proptest`, проверка инвариантов и roundtrip.

## 5. Правила качества (gate перед «готово»)

```bash
just fmt-check    # cargo fmt --all -- --check
just lint         # cargo clippy --workspace --all-targets -- -D warnings
just test         # cargo test --workspace --all-features
# одной командой:
just ci
```

Без `--no-verify`, без `#[allow(...)]` без объяснения в коде.

## 6. Логирование

- Никаких `println!`/`eprintln!` в прод-коде (кроме явного вывода результата по требованию задания).
- В CLI первая строка — `shared::logging::init()?`.
- Структурированные поля: `tracing::info!(bits = n, "generating prime");`.
- `debug!` для внутренних шагов алгоритма (раунд Миллера-Рабина, индекс совпадения).
- Никогда не логируем приватные ключи целиком.
