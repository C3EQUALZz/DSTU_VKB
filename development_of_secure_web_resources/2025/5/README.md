# Лабораторная работа №5. Браузерные события

**Дисциплина:** Разработка защищённых WEB-ресурсов
**Тема:** обработка событий DOM (mouse, keyboard, form, drag & drop, таймеры).

## Цель работы

Изучить механизмы обработки событий в веб-браузере, освоить базовые
понятия и методы работы с событиями DOM, применить их для создания
интерактивных веб-приложений.

## Краткая теория

### Модель событий DOM

- **Capturing phase** (фаза погружения): событие идёт от `window` к цели.
- **Target phase** (фаза цели): событие достигает `event.target`.
- **Bubbling phase** (фаза всплытия): событие поднимается обратно к `window`.

```js
element.addEventListener('click', handler, options);
// options.capture = true → перехват в фазе погружения (редко)
// event.target — реальный элемент-источник
// event.currentTarget — элемент, на котором висит handler
// event.stopPropagation() — остановить всплытие
// event.preventDefault() — отменить действие по умолчанию (например, submit формы)
```

### Делегирование

Один обработчик на родителе вместо N на потомках:

```js
table.addEventListener('click', e => {
    const cell = e.target.closest('[data-field="qty"]');
    if (!cell) return;
    // ...
});
```

Используется в заданиях 5 и 7 — обработчики висят на таблице/контейнере,
а внутри определяется источник.

### Drag & Drop API

```
dragstart → dragenter → dragover → drop → dragend
```

- `event.dataTransfer.setData('text/plain', value)` — передать данные;
- `event.preventDefault()` в `dragover` — обязательно, иначе `drop` не сработает;
- `event.dataTransfer.dropEffect = 'move'` — визуальная подсказка курсора.

## Состав работы (7 заданий)

### Задания 1, 2, 3 — на одной странице

Файл: [`01-nav-modal-timer.html`](01-nav-modal-timer.html),
CSS: [`css/01-nav-modal-timer.css`](css/01-nav-modal-timer.css),
JS: [`js/01-nav-modal-timer.js`](js/01-nav-modal-timer.js).

**Задание 1** — навигация с hover-эффектами. Реализован CSS-`transition`:
`transform: scale(1.2)` + смена цвета фона при `:hover`. JavaScript не
требуется для самих эффектов.

**Задание 2** — модальная форма регистрации. Открывается по клику на
«Зарегистрироваться» в навигации. Валидация:
- email: длина ≤ 254, локальная часть ≤ 64, домен ≤ 255, TLD 2…63,
  ровно один `@`, без двойных точек, разрешённые символы по RFC 5321/6531;
- пароль: ≥ 12 символов, обязательно буквы разного регистра, цифра и
  спецсимвол из `!@#$%^&*`.
- Поле подсвечивается красным, под ним выводится конкретное сообщение.
- Закрытие — крестиком, кнопкой «Закрыть», по `Escape`, кликом по затемнённому фону.

**Задание 3** — таймер обратного отсчёта. Кнопки «Запустить», «Пауза»,
«Сброс». `setInterval(1000)`. Во время работы input заблокирован.
При достижении нуля — сообщение «Время вышло!».

### Задание 4. Галерея с превью

[`04-gallery.html`](04-gallery.html) + JS/CSS.
Использует `mouseenter`/`mouseleave`/`mousemove`: при наведении —
клонирование SVG в превью, при `mousemove` — позиционирование рядом с
курсором (с автоматическим «зеркалированием» у краёв экрана).

### Задание 5. Интерактивная таблица товаров

[`05-products-table.html`](05-products-table.html) + JS/CSS.
- **Одиночный клик** по «Количество» — inline-редактирование
  (`contenteditable="true"`), на `blur` или Enter — пересчёт суммы.
- **Двойной клик** (`dblclick`) по «Цена за единицу» — модальное окно с
  числовым полем.
- **Правый клик** (`contextmenu`) по строке — контекстное меню «Удалить /
  Изменить название / Добавить комментарий». Меню позиционируется у
  курсора, закрывается кликом снаружи или `Escape`.
- **Drag & Drop** строк — `dragstart` / `dragover` / `drop` / `dragend`
  для перестановки.
- Делегирование событий — обработчики висят на `tbody`.

### Задание 6. Всплывающее окно помощи

[`06-help-popup.html`](06-help-popup.html) + JS/CSS.
- Появляется автоматически через `setTimeout(..., 5000)` после
  `DOMContentLoaded`.
- Поле «Ваш вопрос» (textarea), кнопка «Отправить».
- На отправку — `event.preventDefault()`, показ «Спасибо!», очистка
  формы, автозакрытие через 1.5 сек.
- Закрытие — крестиком, кликом по фону, `Escape`.

### Задание 7. Drag & Drop списков студентов

[`07-students-dnd.html`](07-students-dnd.html) + JS/CSS.
- Два списка: «Группа ВИ-31» (с полем «Добавить») и «Задолжники»
  (с плейсхолдером «Перетяни сюда»).
- Карточки автоматически сортируются по алфавиту (русская локаль)
  при каждом изменении.
- Правый клик по карточке → меню «Удалить / Закрыть» справа от курсора
  (с авто-сдвигом, если меню не помещается).
- Drag & Drop **между** списками. При успешном дропе — анимация
  `is-success`. Если бросили вне зоны (`dragend` без `drop`) — анимация
  `is-error` (shake + красная подсветка).
- Счётчики «Всего N» в шапках обновляются автоматически.
- Лимит — 35 карточек суммарно.

**⚠️ По условию задания 7 в CSS нельзя использовать псевдоклассы
(`:hover`, `:active`, `:focus` и т. д.) и псевдоэлементы (`::before`,
`::after`).** Все эффекты «наведения», подсветки drop-зоны, ошибки и
успеха реализованы **через JS**: на `mouseenter`/`mouseleave` /
`dragenter`/`dragleave`/`dragend` JavaScript добавляет/снимает классы
`.is-hover`, `.is-dragover`, `.is-success`, `.is-error`. Сами эти
классы — обычные селекторы, не псевдоклассы.

## Выводы

В работе закреплены ключевые типы DOM-событий — мышиные (`click`,
`dblclick`, `contextmenu`, `mouseenter`/`mouseleave`/`mousemove`),
клавиатурные (`keydown` для `Escape` и `Enter`), drag-and-drop
(`dragstart`/`dragover`/`drop`/`dragend`), события формы (`submit`,
`blur`, `input`) и таймерные (`setInterval`/`setTimeout` поверх
`DOMContentLoaded`). Делегирование позволило обойтись минимальным
числом обработчиков. Для задания 7 продемонстрировано, как полностью
заменить CSS-псевдоклассы JavaScript-эффектами через классы состояний.
