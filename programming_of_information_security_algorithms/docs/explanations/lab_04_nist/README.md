# Лаб 4 — собственная реализация одного из тестов NIST SP 800-22

**Crate:** `crates/lab_04_nist`
**Статус:** ⏳ в разработке

## Условие

> Из стандарта NIST выбрать одну из проверок на выбор и самостоятельно её реализовать.
> С помощью реализованного алгоритма проверить полученную ранее в лабораторной работе №3 последовательность.
> Результат проверки сохранить в файл.

## Выбранный тест: **Frequency (Monobit) Test** (NIST SP 800-22, §2.1)

Самый простой и наглядный тест:

1. Пусть $S_n = \sum_{i=1}^{n} (2 b_i - 1)$ — сумма по преобразованным битам ($0 \to -1$, $1 \to +1$).
2. Вычисляем $s_{obs} = |S_n| / \sqrt{n}$.
3. P-value $= \mathrm{erfc}(s_{obs} / \sqrt{2})$.
4. Если P-value $\geq 0{,}01$ — последовательность считается случайной.

Также реализуем **Runs Test** (§2.3) как дополнительную проверку: смотрит на количество смен 0↔1.

> Можно остановиться на Frequency — это то, что строго требует условие («одну из проверок»). Runs — бонус.

## Реализация (план)

- `domain::tests::monobit::run(&[bool]) -> NistTestResult { p_value, decision }`.
- `domain::tests::runs::run(&[bool]) -> NistTestResult`.
- `domain::erfc` — комплементарная функция ошибок через Abramowitz-Stegun 7.1.26 (полином 6-й степени, погрешность ≤ 1.5·10⁻⁷). Без `libm`.
- `application::analyse` — принимает путь к ASCII-битам из лаб 3 (или бинарному файлу), парсит, прогоняет тест, формирует отчёт.
- `infrastructure::report` — пишет `report.txt` в формате:
  ```
  NIST SP 800-22 §2.1 Monobit Frequency Test
  ─────────────────────────────────────────
  n = 12800
  ones = 6403, zeros = 6397
  S_n = 6
  s_obs = 0.05303
  P-value = 0.95773
  Verdict: RANDOM (P-value ≥ 0.01)
  ```
- `presentation::cli` — `check --input artifacts/lab_03/sequence.bits --out artifacts/lab_04/report.txt`.

## Как протестировать

```bash
cargo test -p lab_04_nist
```

- Контрольный пример из приложения NIST SP 800-22 (§2.1.8): последовательность $\epsilon =$ `1011010101...` (n=100) → ожидаемый p-value $\approx 0.109599$.
- Тест на чистый 0-вектор должен показать P-value ≈ 0 и `Verdict: NON-RANDOM`.
- Тест на чередующуюся последовательность (0101...) для Monobit покажет P-value высокий (≈1), для Runs — низкий.
- Сверка `erfc(0) = 1`, `erfc(∞) = 0`, симметрия `erfc(-x) = 2 - erfc(x)` с приемлемой погрешностью.
