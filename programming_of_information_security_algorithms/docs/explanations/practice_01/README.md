# Практика 1 — длинная арифметика (≥ 64 разрядов)

**Crate:** `crates/practice_01`
**Статус:** ✅ готова — **50/50 тестов** (29 unit+property, 12 векторных на ≥64 разрядов, 9 интеграционных CLI), clippy clean, fmt clean

## Условие

> Необходимо сложить и вычесть два числа, которые по размеру должны быть от 64-разрядов (необязательно одинаковое число разрядов) (можно использовать перенос бита).
> Разрешённые языки: C, C++, Assembler, C# (без сторонних библиотек), можно использовать и другие языки, но так же без использования библиотек.
> По выполнению: при сложении и вычитании должно учитываться, что разряд числа может меняться (для наглядности можно ввести кол-во разрядов); вывести проверку обычного сложения и вычитания (то есть не через программу, а через встроенное сложение и вычитание).

Источник: `docs/conditions/1 Практика.docx`.

## Что считается «разрядом»

В задании речь о **разрядах числа** (digit) — позициях в десятичной записи. «От 64 разрядов» = число содержит ≥ 64 десятичных цифры (это сильно больше, чем помещается в `u64` / `u128`).

Поэтому без длинной арифметики не обойтись: пишем свой `BigInt` поверх вектора 32-битных limb'ов, и сравниваем результат с встроенным сложением/вычитанием в `i128` **для маленьких сценариев**, где встроенный тип ещё может хранить число (это и есть «проверка обычного сложения», как просит условие).

## Реализация (план)

- `domain::bigint::BigInt` — знаковое целое произвольного размера.
- `domain::bigint::add` / `domain::bigint::sub` — без `std::ops::Add` (чтобы не дёргать перегрузку, а явно показать алгоритм с переносом/займом).
- `application::verify` — сценарий: парсит две строки, выполняет сложение/вычитание через `BigInt`, и, если влезает в `i128`, — повторяет операцию встроенной арифметикой и сверяет с результатом.
- `presentation::cli` — `clap` с подкомандами `add A B`, `sub A B`, `demo` (генерирует случайные большие числа и показывает таблицу).

## Как запустить

```bash
just practice-01 add 123456789012345678901234567890 987654321098765432109876543210
just practice-01 sub 12345678901234567890 98765432109876543210
just practice-01 demo
```

Уровень логов:

```bash
RUST_LOG=debug just practice-01 add 12345...
```

## Что должно быть видно

- При операциях с маленькими числами CLI выводит **рядом** результат своего `BigInt` и встроенного `i128` — они должны совпадать (это и есть «проверка обычного сложения/вычитания»).
- При больших числах (>38 десятичных цифр) `i128` не справится — CLI явно говорит «встроенный тип переполнен, проверки не было», а `BigInt` продолжает работать.
- Изменение разрядности (перенос/займ через границу) логируется на уровне `debug`.

### Пример реального вывода

```

Проверка:  OK ✓ оба пути совпадают (встроенная арифметика согласна)

```
