# КДБ8 — метод Куттера-Джордана-Боссена в Mathcad Prime

## Что открыть
- **`kdb8_labeled.mcdx`** — рабочий лист с русскими подписями и описаниями шагов (для отчёта).
- `kdb8.mcdx` — тот же лист без подписей (компактнее).
- Рядом с `.mcdx` **обязательно** должен лежать `container.bmp` — лист читает его функциями `READRGB / READ_RED / READ_GREEN / READ_BLUE`. Все файлы уже в одной папке; если переносите лист — переносите и `container.bmp`.

## Порядок в Prime
1. Открыть `kdb8_labeled.mcdx`.
2. Пересчёт: **Расчёт → Пересчитать всё** (Ctrl+F5). Лист тяжёлый: N = τ·Lm·8 = **3600** операций встраивания и столько же извлечений — пересчёт может занять **минуты**. Это нормально.
3. Внизу листа проверить: `Recovered = "Pierre did not choose a career"`, `err = 0`.

## Параметры (вариант)
`M = "Pierre did not choose a career"`, `v = 0.15`, `τ = 15`, `Rr = 4`, `σ = 3`, `Ko = 125` → ключ `K = [125, 156, 243, 59, 34, 115, 132, 174]`.

## Если Prime считает слишком долго
Логика проверена и на облегчённой конфигурации `M = "Pierre"`, `τ = 15` (N = 720, err = 0). Скажите — перегенерирую лёгкий лист за минуту (`gen_mcdx.py`).

## Как всё сгенерировано (для себя)
- `kjb_sim.py` — эталонная Python-симуляция той же математики (доказывает err = 0 до Prime).
- `gen_assets.py` — генерирует `container.bmp`, `stego.bmp`, `message.bmp` (+ png для отчёта).
- `gen_mcdx.py` — собирает `worksheet.xml` и пакует `.mcdx` (`LABELS=1` — вариант с подписями).
- Отчёт: `scripts/kutter_jordan_bossen/build.py` → `docs/reports/2025/8/КДБ8.docx`.

## Заметки по Prime (грабли)
- `WRITERGB(...):=S` из приложения методички в Prime **не работает** — round-trip встраивание→извлечение сделан **в памяти** (`Bb := B'`), файлы не перезаписываются.
- `D2B`/`B2D` определены как функции листа (не полагаемся на встроенные), middle-square посчитан **арифметически** (floor/mod/if) — без `num2str`/`str2num`.
- ORIGIN := 1 (как в приложении), индексы и биты 1-based.
- `vec2str` показывает коды как символы; для латиницы текст читается напрямую.
